# webpack loader-打造一个ignore-loader

# loader是什么

所谓 loader 只是一个导出为函数的 JavaScript 模块。loader runner 会调用这个函数，然后把上一个 loader 产生的结果或者资源文件(resource file)传入进去。函数的 this 上下文将由 webpack 填充，并且 loader runner 具有一些有用方法，可以使 loader 改变为异步调用方式，或者获取 query 参数。

官方文档：https://www.webpackjs.com/api/loaders/

# 作用

我想在项目任何地方写一个开发环境使用的代码，并不会在打包时留下。
思路：

  1. 使用注释包裹
  2. 不会影响其他注释

# 创建ignore-loader

/**
 * @name ignore-loader
 * @description 忽略这块的代码
 * @date 2020-03-19
 * @sample
 * <!-- [start] -->
 * <div>ignore</div>
 * <!-- [end] -->
 * 
 * /* [start] *\/ 记得把转义去掉
 *  code
 * /* [end] *\/
 */
module.exports = function loader(source) {
  const ignoreTemplate = /(<!-- |<!--)\[start]([\s\S]*)(\[end] -->|\[end]-->)/gi;
  const ignoreJsOrCss = /(\/\* \[start]|\/\*[start])([\s\S]*)(\[end] \*\/|\[end]\*\/)/gi;
  source = source.replace(ignoreTemplate, '').replace(ignoreJsOrCss, '');
  return source;
};

# 在vue.config.js(webpack.config.js)中使用

因为我是在vue项目中用到了这个，但是概念差不多，只需要在module下使用

webpack.config.js

// webpack.config.js
module: {
  rules: [ // 规则更多使用请移步官网文档
    {
      test: /\.vue$/, // 匹配所有的.vue文件
      loader: 'ignore-loader'
    }
  ]
}

vue.config.js

// vue.config.js
chainWebpack: config => {
  config.module
    .rule('vue')
    .use('./ignore-loader')
    .loader('./ignore-loader')
    .end();
  config.module
    .rule('js')
    .use('./ignore-loader')
    .loader('./ignore-loader')
    .end();
}

# 结语

配置完webpack之后就可以在项目中使用了，当然请注意区分开发环境和正式环境。如果你想在开发环境中保留的话，那么就需要在配置文件中进行环境处理。
