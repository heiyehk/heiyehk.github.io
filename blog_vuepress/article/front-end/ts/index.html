# 在typescript中使用eslint+prettier

在ts的规范中，主要选择有eslint和tslint。而tslint已经停止维护，官方也声明了在2019 路线图中明确表明后续对 Lint 工具的支持和建设会以对 ESLint 进行适配的方式为主https://github.com/Microsoft/TypeScript/issues/29288

# prettier安装和配置

# 安装

npm i -D prettier eslint-plugin-prettier eslint-config-prettier
1

# 配置

创建.prettierrc.js文件

module.exports =  {
  // 单行长度，默认80
  // "printWidth": 80
  // tab缩进大小,默认为2
  "tabWidth": 2,
  // 使用tab缩进，默认false
  "useTabs": false,
  // 使用分号, 默认true
  "semi": true,
  // 使用单引号, 默认false(在jsx中配置无效, 默认都是双引号)
  "singleQuote": true,
  // 行尾逗号,默认none,可选 none|es5|all
  // es5 包括es5中的数组、对象
  // all 包括函数对象等所有可选
  "trailingComma": "none",
  
  // 对象中的空格 默认true
  // true: { foo: bar }
  // false: {foo: bar}
  "bracketSpacing": true,
  // JSX标签闭合位置 默认false
  // false: <div
  //          className=""
  //          style={{}}
  //       >
  // true: <div
  //          className=""
  //          style={{}} >
  "jsxBracketSameLine": false,
  // 箭头函数参数括号 默认avoid 可选 avoid| always
  // avoid 能省略括号的时候就省略 例如x => x
  // always 总是有括号
  "arrowParens": "avoid"
}

# eslint的安装和配置

# 安装

npm i -D eslint @typescript-eslint/eslint-plugin @typescript-eslint/parser babel-eslint
1

# 配置

module.exports = {
  parser: '@typescript-eslint/parser', // Specifies the ESLint parser
  extends: [
    'plugin:@typescript-eslint/recommended', // Uses the recommended rules from the @typescript-eslint/eslint-plugin
    'prettier/@typescript-eslint', // Uses eslint-config-prettier to disable ESLint rules from @typescript-eslint/eslint-plugin that would conflict with prettier
    'plugin:prettier/recommended', // Enables eslint-plugin-prettier and displays prettier errors as ESLint errors. Make sure this is always the last configuration in the extends array.
    'prettier'
  ],
  plugins: ['@typescript-eslint', 'prettier'],
  env: {
    node: true,
    es6: true,
    mocha: true
  },
  rules: {
    // 0: 关闭 1: 警告 2: 报错
    'quotes': [1, 'single'],
    '@typescript-eslint/explicit-function-return-type': 0,
    'comma-dangle': [1, 'never'],
    'prettier/prettier': 1 // 这里控制prettier规范
  }
};

如果需要配置这个则需要安装全局的eslint

1

在package.json中加入命令

"lint": "eslint src --fix --ext .ts"
